Gran Cantidad de información que se maneja habitualmente está conformada por conjuntos de datos que se modelan como listas, las cuales permiten representar información compuesta, incluso cuando no se tiene presente cuantos elementos se quieren enumerar.

Organizar la información en forma de listas es una tarea realizamos regularmente cuando planificamos una actividad, como preparar un evento, planear un viaje o ir de compras. Ejemplos de estas estructuras son: listas de útiles escolares, listas de los equipos de futbol de Colombia, listas de estudiantes de una institución, listas de productos en un inventario, listas de los empleados de una empresa, listas de chequeo, entre otras.

Una lista se puede definir como un conjunto de elementos que puede ser del mismo o de diferentes tipos de datos, agrupados bajo una misma denominación (nombre de la lista) o sin un nombre determinado.

Siguiente

una lista es un tipo de dato en DrRacket, que se caracteriza por ser dinámica, ya que se le puede añadir información.

Una lista vacía consiste en un elemento especial que no es un símbolo (un identificador) ni una pareja (estructura de datos conformada por dos campos cabeza y cola), pero sí una lista.

Se trata de un objeto atómico que denota una lista vacía.
Siguiente

Una lista vacía se puede escribir de diferentes formas en DrRacket como se muestra a continuación:

Con la constante especial empty
Con la constante especial null
Con la forma tradicional de Lisp: '(), que es un carácter de apóstrofe seguido de paréntesis vacíos.

Implementacion de una Lista vacia en DrRacket

Siguiente
Volver

Para definir una lista se debe asignar un nombre y se crea, ya sea vacía o conformada con los elementos que el programador incluya. La expresión para crear una lista tiene la siguiente forma:

(define nombre-lista ' (elem1 elem2 elem3…elemn))
Nombre-lista

Hace referecia al nombre que el programador le asigna a la lista según su gusto y conveniencia.

Apóstrofe (') o función quote

Indica que lo que sigue entre paréntesis es una lista de valores literales.

Elementos

Donde los elem1 elem2 elem3…elemn son los componentes de la lista. Cada elemento puede ser un dato de
diferente tipo, ya sea numérico, carácter, cadena, símbolo u otra lista.

Introducción

El lenguaje de programación DrRacket posee diferentes funciones que permiten manipular una lista, a través de ellas es posible acceder a una lista, adicionar elementos, unir listas, calcular la longitud, buscar y filtrar datos, entre otras.
Las funciones que se muestran a continuación están predefinidas en DrRacket y pueden ser utilizadas en cualquier parte del programa.

Funciones Car y First

Ambas funciones realizan la misma acción, la cual consiste en retornar el primer elemento de una lista. Las sintaxis de Car y First es la siguiente:

(car lista)

(first lista)

Ejemplo: obtener el primer elemento de la lista colores-primarios definida de la siguiente manera:

(define colores-primarios ( cons "Rojo" ( cons "Amarillo" ( cons "Azul" empty) ) ) )

Al aplicar estas funciones a una lista vacía se genera error de ejecución, debido a que una lista vacía no posee elementos.

Funciones Cdr y Rest

Cdr y Rest se encargan de retornar el resto o cola de una lista, es decir que devuelven una sublista con los elementos de la lista original sin incluir el primer elemento. La sintaxis de estas funciones es la siguiente:

(cdr lista)
(rest lista)
 Ejemplo 1: Realice un programa que cree una lista de países y de ella obtenga todos los países a excepción del primero.
 Ejemplo 2: Defina una lista con dos elementos y de ella obtenga tanto el primero como el resto de la lista.

Funcion Length

Esta función determina la longitud de una lista, de tal manera que cuenta el número de elementos que la conforman. La sintaxis de la función length es la siguiente:

(length lista)

Ejemplo: Realice una lista de elementos y muestre el número de elementos que la conforman.

Funcion Append

Esta función permite construir una lista en la que se unen listas que se le pasan como argumento. La sintaxis de la función append es la siguiente:

(append lista1 lista2...)

Ejemplo: Construya tres listas que contengan diferentes elementos, luego defina una lista que almacene todos los elementos de las listas previamente definidas.

Funcion Reverse

Esta función permite invertir el orden de una lista. La sintaxis de la función reverse es la siguiente:

(reverse lista)

Ejemplo: Dada la siguiente lista, invierta el orden de sus elementos.

(define listaNumeros (cons "cero" (cons 1 (cons "dos" (cons 3 (cons 5.322 empty ))))))

Funcion Member

Permite corroborar si un elemento pertenece a una lista determinada. La sintaxis de la función member es la siguiente:

(member elemento-a-buscar lista)

  • Definición

    La recursividad es el proceso mediante el cual una función puede llamarse a sí misma desde la misma función, de tal manera que se ejecuta una cantidad finita de veces. De acuerdo a esto, una de las partes más pequeñas de la función puede ser la misma función que se esté tratando de implementar.

    CASOS BASE

    Son aquellos casos que se resuelven sin recursividad, y se constituyen condiciones de parada para que la recursividad no sea infinita. Este tipo de casos debe proporcionar una solución o acercar al programador a ella.

    CASOS RECURSIVOS

    Son aquellos que siempre deben avanzar hacia un caso base, y son considerados expresiones de forma recursiva.
